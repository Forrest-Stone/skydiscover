# Budget Aggregation & Speedup Analysis (Detailed Guide)

This README explains, step-by-step, how to:

1. run search methods and collect budget traces,
2. check whether per-run plots were generated automatically,
3. run `scripts/plot_budget_curves.py` to generate aggregate CSV/PNG figures,
4. troubleshoot common "no images generated" issues.

It is designed so you can **save statistics first**, and then **re-generate plots later** at any time.

---

## 1) Prerequisite: each run must emit budget artifacts

To aggregate and compare methods, each run directory must include at least:

- `summary.json`
- `iterations.jsonl`

The aggregation script discovers runs by scanning `summary.json` under `--root`
and reading sibling `iterations.jsonl`.

### Per-run plots generated by controller (during/after run)

If matplotlib is available, run completion should also produce per-run images such as:

- `best_score_vs_cost.png`
- `best_so_far_objective_vs_cost.png`
- `best_so_far_combined_score_vs_cost.png`
- `budget_report.png`

If any are missing, check `budget_plot_status.txt` in the same run directory.

Typical output layout:

```text
outputs/
  costada/
    <task_or_run_name>/
      summary.json
      iterations.jsonl
  adaevolve/
    <task_or_run_name>/
      summary.json
      iterations.jsonl
  evox/
    <task_or_run_name>/
      summary.json
      iterations.jsonl
```

If a run has no `summary.json`, it is skipped.

## 2) Run aggregation script (metrics + CSV + plots)

### Minimal command

```bash
python scripts/plot_budget_curves.py --root outputs --out-dir outputs/aggregate
```

### Full command (recommended)

```bash
python scripts/plot_budget_curves.py \
  --root outputs \
  --out-dir outputs/aggregate \
  --budgets 0.5,1.0,2.0 \
  --target 0.92 \
  --baselines adaevolve,evox
```

### Main arguments

- `--root`: root directory containing all run outputs.
- `--out-dir`: where aggregated CSV/plots are written.
- `--budgets`: budget grid for sliced metrics (comma-separated).
- `--target`: target threshold `tau` for `Success@Target` and `Cost-to-Target`.
- `--baselines`: one or multiple baselines for pairwise speedup.

### What you should see in terminal

When plots are generated successfully, the script prints lines like:

- `Wrote: outputs/aggregate/best_so_far_objective_vs_cost.png`
- `Wrote: outputs/aggregate/oob_rate_vs_budget.png`
- `Wrote: outputs/aggregate/speedup_vs_baseline.png`

If matplotlib is unavailable, it prints:

- `Skipped plotting (matplotlib unavailable).`

- `all_iterations.csv`

## 4) Speedup@Target definition and computation

For method `m`, baseline `b`, and target `tau`:

\[
\mathrm{Speedup@Target}(\tau; m, b)
=
\frac{\mathrm{median}(\mathrm{CostToTarget}_b(\tau))}
{\mathrm{median}(\mathrm{CostToTarget}_m(\tau))}
\]

Implementation rules:

1. Use **successful runs only** (`Cost-to-Target` exists).
2. Use **median**, not mean.
3. If baseline side or method side has no valid successful runs, speedup is `NaN`.
4. Baseline is generic (not hardcoded).

---

## 5) Required columns expectation for speedup

The script can derive speedup when run-level/per-run metrics include:

- `method`
- `task_family`
- `task_name`
- `nominal_budget`
- `target`
- `success_target`
- `cost_to_target` (run-level) / `Cost-to-Target` (per-run metrics)

These are generated by the current script output path.

---

## 6) Practical workflow

1. Run each method (`costada`, `adaevolve`, `evox`, ...), ensure each run writes `summary.json` + `iterations.jsonl`.
2. Run aggregation once over the shared `outputs/` root.
3. Read:
   - `aggregate_metrics.csv` for primary budget metrics.
   - `speedup_vs_baseline.csv` for pairwise efficiency headlines.
4. Use PNGs for quick sanity-check/appendix visualization.

---

## 7) Troubleshooting checklist

If no images are generated, check in this order:

1. **Does each run have `summary.json` and `iterations.jsonl`?**
   - If missing, aggregation cannot proceed.
2. **Is matplotlib installed in your runtime?**
   - Without it, plotting is skipped by design.
3. **Do iteration rows contain budget fields (e.g., `cumulative_cost`)?**
   - Empty/invalid traces cause plot functions to return `False`.
4. **Did you point `--root` to the correct directory?**
   - The script scans recursively from `--root`.
5. **Check controller note file**:
   - `budget_plot_status.txt` in run directory for per-run plot failures.

---

## 8) Re-generate plots later (no re-run required)

As long as run directories still contain `iterations.jsonl` and `summary.json`,
you can regenerate aggregate plots anytime:

```bash
python scripts/plot_budget_curves.py \
  --root outputs \
  --out-dir outputs/aggregate_rerun \
  --budgets 0.5,1.0,2.0 \
  --target 0.92 \
  --baselines adaevolve,evox
```
